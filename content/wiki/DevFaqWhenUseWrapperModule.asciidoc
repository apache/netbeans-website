<div class="sect1">
<h2 id="_when_should_i_use_a_library_wrapper_module_and_when_should_i_just_package_the_library_into_my_module">When should I use a library wrapper module and when should I just package the library into my module?</h2>
<div class="sectionbody">
<div class="paragraph">
<p>The New Module Wizard offers easy support for creating a wrapper module:
<strong>File &gt; New Project &gt; NetBeans Modules &gt; Library Wrapper Module</strong> and since NetBeans 6.8 it is similarly easy to either edit Library Wrapper Module after it has been created or package library directly to your module via <strong>Project Properties &gt; Libraries &gt; Wrapped JARs</strong>.</p>
</div>
<div class="paragraph">
<p>Before NB 6.8 it was more convenient to create Library Wrapper module due to existence of the wizard, but not Project Properties UI. This biased the answer to this question, but generally there&#8217;s no harm in using a library wrapper module.</p>
</div>
<div class="paragraph">
<p>Note that a library wrapper module can wrap more than one external JAR -
you do not need to create one for each library.
But it is a good idea to create a separate wrapper for each JAR
if they come from different projects and might conceivably be used independently.</p>
</div>
<div class="paragraph">
<p>The general algorithm for making an optimal decision about when to use a wrapper module is this:</p>
</div>
<div class="ulist">
<ul>
<li>
<p>If the library is from some third-party source, and your module is just using it, then</p>
</li>
<li>
<p>If it is something very esoteric and your module will be the <em>only</em> module in an installation of NetBeans (or your NetBeans-based app) ever using it</p>
</li>
<li>
<p>You are writing only one module that will use the library, <em>or</em> you want to declare the packages contained in your library in the public packages of your module</p>
</li>
<li>
<p>If you will never want  deliver an update of that library by itself, without delivering an update of your module</p>
</li>
<li>
<p>Then you probably just want to package the library directly into your module, as in <a href="DevFaqModuleDependencies.asciidoc">DevFaqModuleDependencies</a></p>
</li>
<li>
<p>If you wrote the library you want to use</p>
</li>
<li>
<p>You might just want to add the appropriate <code>OpenIDE-Module-*</code> entries directly to its manifest and make it a module that way.  Remember to list the packages your module will need to export.</p>
</li>
<li>
<p>Else, you probably want to read <a href="DevFaqWrapperModules.asciidoc">DevFaqWrapperModules</a></p>
</li>
</ul>
</div>
<div class="paragraph">
<p>There is a very slight performance penalty to using a wrapper module -
it&#8217;s one more JAR to open and read from, and one extra layer of indirection for the classloader.
<em>That is not a reason to avoid using a wrapper module if that&#8217;s what you need</em> -
it really is slight.
In a very large application such as the NetBeans IDE,
such considerations are more important because there are more JARs, more classloaders,
and hence more overhead already.</p>
</div>
<div class="paragraph">
<p>If you are developing the library yourself, but decide you want to keep the library project separate from any NB module project, you can do so.
See <a href="DevFaqWrapperModules.asciidoc">Using a wrapper module for an existing project</a>
for information and various methods to hook them up for
development.</p>
</div>
<div class="paragraph">
<p>Applies to: NetBeans 6.8 and above</p>
</div>
</div>
</div>
