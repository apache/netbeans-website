<div class="sect2">
<h3 id="_installing_things_declaratively_as_opposed_to_programmatically">Installing things declaratively as opposed to programmatically</h3>
<div class="paragraph">
<p>Long ago, in a galaxy far far away, most objects in NetBeans were created during startup.  That works fine for a small application.  It is disaster for a large application - with each new component in the system, startup time gets longer, garbage collections get more frequent and memory requirements rise.</p>
</div>
<div class="paragraph">
<p>Because of this, today, most of the APIs you will use to install things into the the IDE or a <a href="http://platform.netbeans.org/">NetBeans Platform-based application</a> involve a text entry of some sort,
such as <a href="DevFaqModulesLayerFile.asciidoc">putting something in an XML file</a>), or a plain-text file naming some classes in your JAR file&#8217;s <code>META-INF/services</code> directory &mdash; <em>not</em> running Java code.
<strong>A well-behaved module should do nothing on startup.</strong></p>
</div>
<div class="paragraph">
<p>The main ways to accomplish this goal are to use either the [<a href="http://bits.netbeans.org/dev/javadoc/org-openide-util-lookup/org/openide/util/lookup/ServiceProvider.html">http://bits.netbeans.org/dev/javadoc/org-openide-util-lookup/org/openide/util/lookup/ServiceProvider.html</a> <code>@ServiceProvider</code>] annotation (which generates the aforementioned <code>META-INF/services</code> files at compile-time), or by registering files or objects in your module&#8217;s <a href="DevFaqModulesLayerFile.asciidoc">XML layer file</a>.  As of NetBeans 6.9, there are <a href="http://wiki.apidesign.org/wiki/CompileTimeCache">a number of annotations</a> which can be used to generate the this XML data at compile-time.</p>
</div>
<div class="paragraph">
<p>Using declarative mechanisms, your objects will be instantiated when they are needed to do actual work.  Until something really needs to call them, they don&#8217;t need to be created and their classes are not loaded.</p>
</div>
<div class="paragraph">
<p>If you really need to run some code on startup, create a subclass of <code><a href="http://bits.netbeans.org/dev/javadoc/org-openide-modules/org/openide/modules/ModuleInstall.html">ModuleInstall</a></code></p>
</div>
<div class="paragraph">
<p><a href="DevFaqWhenToUseWhatRegistrationMethod.asciidoc">More about registration mechanisms</a>
&lt;hr/&gt;</p>
</div>
<div class="paragraph">
<p>Applies to: NetBeans 6.7 and later</p>
</div>
</div>
